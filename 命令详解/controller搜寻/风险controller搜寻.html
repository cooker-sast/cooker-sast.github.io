支持两种形式的风险controller搜寻：

  1. 基于 invoked-chain 从危险方法出发，筛选哪些controller调用了这些方法
  2. 基于 call-chain 首先搜集全量的controller，然后从每一个controller出发，看看能不能触发危险方法。

# 运行（Run）

java -jar cooker.jar vuln-controller --acp java-sec-code-1.0.0.jar --work-dir vuln-controllers --ic true --cc false
1

用ic不用cc

# 结果（Result）

file 说明 case
sink-entry-method-file.yml 危险方法的集合，作为ic的start method file
controller_methods.yml 所有最终能够触发危险方法的controller的集合
ic目录 ic目录下面就是invoked-chain分析的结果，具体参考反向方法调用寻径

如果用了cc，就会生成一个cc目录，其中的文件就是call chain分析的结果，具体参考正向方法调用寻径，然后找到的controller也会被加入controller_methods.yml里面。
