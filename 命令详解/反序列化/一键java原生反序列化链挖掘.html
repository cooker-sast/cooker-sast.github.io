jnd gadget chain的挖掘用到的也是污点分析算法，只不过由于场景的特殊性，一个对象一旦是污点数据，往往意味着这个对象的类和对象的所有属性都是攻击者完全可控的，而且对于环境相关的类，比如jre中的类和tomcat中的类，的方法调用，在gadget chain挖掘的场景中也几乎不能忽视。

# 运行（Run entry search）

这里以挖掘cc链为例。

java -jar cooker.jar jnd-gadget --acp commons-collections-3.1.jar;C:\Users\inhann\.cooker\JREs\jre1.8\rt.jar --work-dir gadget --worker 30 --time 60 --entry-search
1

--entry-search 用来搜寻满足这样条件的 sink： base 可控，且arg可控

# 结果（Result entry search）

file 说明 case
entry-methods.yml 所有集成了Serializable的readObject
sink.txt 触发的所有sink，比如其中的 <java.lang.Object: boolean equals(java.lang.Object)>/entry 就意味着能够从某个readObject出发，调用任意一个对象(base)的equals()方法，且两个参数(arg)也可以是任意对象
gx/complete 调用栈，完整
gx/simple 调用栈，简单。case表示从javax.management.BadAttributeValueExpException 的 readObject 出发可以调用任意对象的 toString 方法。

# 运行（Run gadget search）

java -jar cooker.jar jnd-gadget --acp commons-collections-3.1.jar;C:\Users\inhann\.cooker\JREs\jre1.8\rt.jar --work-dir gadget2 --worker 30 --time 60 --entry-method-feature-yaml-input toString-feature.yaml
1

其中 toString-feature.yaml 的内容为：

subclassOf:
     - java.io.Serializable
classNamePattern: null
methodNamePattern: null
methodNamePatternCaseSensitive: true
methodSubsig:
     - java.lang.String toString()
invokeMethod: null
paramType: null
paramCount: null
allowPhantom: false
