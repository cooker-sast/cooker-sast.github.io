# 运行（Run）

java -jar cooker.jar webvuln --acp java-sec-code-1.0.0.jar --work-dir vuln --worker 30 --ic
1

--ic 表示用 invoked-chain 进行搜寻风险controller，默认是对全量controller做扫描 --worker 表示并发量，如果不加--ic 就是分析所有controller

# 结果（Result）

1. 对于漏洞挖掘场景而言，可以先查看 ta/sink.txt，这里面是所有能触发的危险方法，然后可以在ta/complete或者ta/simple中找到调用了危险方法的controller及其调用链路，通过 ta/flow 可以看到流入危险方法的关键参数的污点数据的来源和流动路径。最后结合上述的结果，进行人工复查和调试，确定是否存在漏洞。

2. 也可以查看ic的结果，看看哪些危险方法目录下面有methods.yml文件，然后去查看对应path目录下面的从controller到危险方法的调用路径，进行人工复查和调试，确定漏洞。

file 说明 case
ic vuln-controller 使用 invoked-chain来搜寻风险controller，而ic就是其结果，具体参考反向方法调用寻径
ta ta是污点分析的结果，具体参考污点分析。
all-controller-methods.yml 所有找到的controller，也是污点分析的入口
sink-entry-method-file.yml 危险方法的集合，作为ic的start method file
vuln-controller-methods.yaml 所有找到的风险controller

ic 之下有多个目录，每个目录对应一个evil method的结果： 以 java.lang.Runtime##java.lang.Process#exec(java.lang.String) 目录下的内容为例

file 说明 case
methods.txt 里面是分析得到的所有 <org.joychou.controller.Rce: java.lang.String CommandExec(java.lang.String)> 能够直接过间接调用到的方法
relation.json 里面是分析得到的所有 <org.joychou.controller.Rce: java.lang.String CommandExec(java.lang.String)> 出发的调用关系。开头项是子项的调用者。
methods.csv 通过处理methods.txt得到，主要为了方便导入neo4j
relation.csv 记录的是relation的csv，为了方便导入neo4j和进行路径搜索的图算法
methods.yaml 可以认为是methods.txt里面的method经过method search筛选过的结果
paths 起点函数和筛选得到的method之间的路径

ta 是污点分析的结果

file 说明 case
simple 方法调用路径。simple下每一个目录对应的是一个controller的分析结果，每一个文件的文件名代表最终调用危险方法，文件内容是方法调用栈
complete 和simple的结构基本一致，文件内容除了方法调用栈的方法之外，还记录了调用的具体callsite。可以理解为记录了调用时的具体语句，但是这个调用语句的形式是IR，如果想查看这个调用语句在整个类中的具体位置可以把对应类的IR给dump下来研究，方法参考导出ir
flow taint flow，展示污点的流动情况，是一个 list of 变量，表示污点数据是从最上面的变量流到最下面的变量的。污点数据是使用者可控的数据。
sink.txt 分析结束之后整理所有controller都触发了哪些sink

一般情况下，我们可以通过flow的内容查看是哪个来自用户的输入最终跑到了危险方法的参数当中

通过complete或者simple的内容查看方法的调用栈。

complete和flow涉及的callsite和csvar的格式是tai-e ir，如果想要进一步理解结果和调试，可以dump出对应class的ir进行查看，具体方法参考导出ir

# --controller-method-feature-yaml-input

name 说明 required default
--controller-method-feature-yaml-input controller 的样式 false

给 vuln-controller 用

# --controller-method-file-yaml-input

name 说明 required default
--controller-method-file-yaml-input controller yaml文件，作为污点分析的入口 false

考虑到controller搜寻和污点分析有时候可能并不是一条龙进行的

# --taint-config

name 说明 required default
--taint-config taint config的位置 false

默认用的 $HOME/.cooker/taint-config/default.yml taint config 的具体说明参考 污点分析

# --time1

name 说明 required default
--time1 ic 运行的time limit false 60

给 vuln-controller 用

# --time

name 说明 required default
--time 每一个全程序分析的最大用时 false 60

如果是-1表示没有时间限制。用于污点分析。

# --level

name 说明 required default
--level 方法调用链路的最大长度 false 7

controller的搜寻还有污点分析都会用到

# --ic

name 说明 required default
--ic 使用invoked chain分析来找controller false true
