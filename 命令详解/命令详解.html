subcommand 说明
method 具体参考method搜寻。筛选和搜集特定的method
class 具体参考class搜寻。筛选和搜集特定的method
invoked-chain 具体参考反向方法调用寻径。反向方法调用寻径，一般用于从危险函数出发，寻找其间接调用者，最终生成调用关系的集合
call-chain 具体参考正向方法调用寻径。按照程序运行所调用方法的顺序，从某一个特定方法触发，生成调用关系的集合
controller 具体参考全量controller搜寻。搜寻所有controller
vuln-controller 具体参考风险controller搜寻。搜寻最终可能触发危险方法的controller
taint 具体参考污点分析。k-call context sensitive的指针分析，更契合于漏洞挖掘实践的污点分析
webvuln 具体参考一键web漏洞扫描。顾名思义，直接对一个web应用做漏洞扫描
gadget 具体参考GadgetChain一般挖掘。基于污点分析的反序列化利用链的挖掘方案
jnd-gadget 具体参考一键java原生反序列化链挖掘。顾名思义，一键挖java原生反序列化利用链
ir 具体参考导出ir以文件的形式dump ir，方便调试和理解污点分析的结果
decompile 反编译，用cfr，具体参考decompile
semgrep 使用semgrep分析非java的代码和文本，具体参考semgrep

下文是作用于全局的参数的说明。

# --acp

name 说明 required default
--acp application class path，指定java程序的位置 true

cooker会根据acp的值处理java程序，以下是cooker能够处理的java程序的类型：

type pattern 处理方式
war 如果acp以.war结尾，则认为是一个 war 文件 会unzip成一个以temp_war_unzip_开头的目录，作为exploded war处理
exploded war 如果acp是一个目录，而且WEB-INF在目录中，则认为是exploded war /WEB-INF/classes 中的所有 class 、/WEB-INF/lib 中的所有 jar，都会被加入world当中。特别的，/WEB-INF/classes 中的所有 jsp都会被编译成class加入world当中
springboot fatjar 如果acp以.jar结尾，gx会判断其中是否有BOOT-INF 目录，如果有，则认为是springboot fatjar 会unzip成一个以temp_fat_unzip_开头的目录，然后/BOOT-INF/classes中所有的class、/BOOT-INF/lib下所有的jar都会被加入world
dependency jar 如果acp以.jar结尾，又不是fatjar，则会被任意是一个普通的jar 直接加入world
list of jars 如果acp是一个目录，又不是 exploded war，其中还有一个list.txt 则认为这个目录是 list of jars 将这个目录下的所有jar都加入world
exploded dependency jar 如果acp是一个目录，不是 exploded war ，也不是list of jars ，则认为是一个exploded dependency jar 直接加入world
其他 直接加入world

# --resources-dir

name 说明 required default
--resources-dir 规则文件的位置 false ""

一旦--resources-dir 为空，就会赋值为 $HOME/.cooker/ ，如果cooker找不到资源文件就会报错。资源文件当中包括部分分析要用到的规则，以及加入acp做分析的jre。资源文件地址 (opens new window)

# --wc

name 说明 required default
-wc 是否缓存world false true

考虑到有的程序可能会经历多次分析，因而cooker在默认情况下开启 world cache，会将程序的中间代码表示进行缓存，默认情况下，缓存文件位于 --world-cache-dir 指定的目录

# --world-cache-dir

name 说明 required default
--world-cache-dir world缓存存储的位置 false "cache"

如果 world cache开启，则会将world缓存文件放在 --world-cache-dir 之下，默认是资源文件的cache 目录，即$HOME/.cooker/cache

# --world-cache-file

name 说明 required default
--world-cache-file world缓存文件名 false ""

可以是绝对路径或者相对路径，如果指定了--world-cache-file，那cooker就会直接读取该文件进行处理

# --include-lib

name 说明 required default
--include-lib 是否将lib加入world false true

这个参数主要是用于acp为war、exploded war、springboot fatjar的情况，如果是true，则lib下面的所有jar就会被加入world，否则就不会加入world。

一般情况下，这个参数没什么用，但是有一些分析中不需要考虑lib，--include-lib为false的话，会分析得快一点。

# --work-dir

name 说明 required default
--work-dir 工作目录 false

分析结果的输出目录

# --worker

name 说明 required default
--worker 工作目录 false 30

并发运行的分析的最大数量，默认是30并发。
