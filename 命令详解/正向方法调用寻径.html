正向方法调用寻径，一般用于从controller出发，最终生成调用关系的集合，可以根据这个集合生成其中任意两个方法之间的调用路径。

# 运行（Run）

java -jar cooker.jar call-chain --acp java-sec-code-1.0.0.jar --entry-method-yaml-input start-controller.yml --dir-output cc-results --worker 10 --time 60 --level 10 --method-feature-yaml-input evil-mff1.yml+evil-mff2.yml+evil-mff3.yml --work-dir .
1

start-controller.yml内容如下：

---  
entryMethods:  
  - entryClass: "org.joychou.controller.Rce"  
    methodSig: "java.lang.String CommandExec(java.lang.String)"

以上命令能够从start method，即 <org.joychou.controller.Rce: java.lang.String CommandExec(java.lang.String)> 出发，构造方法调用图，最终在所有能够触及的方法当中判断是否有与命令执行相关的方法，结果保存在cc-results目录。level 为7意味着最长的调用链路是10，time为60意味着从每一个 start method 开始的搜寻只会进行60秒。

# 结果（Result）

file 说明 case
methods.txt 里面是分析得到的所有 <org.joychou.controller.Rce: java.lang.String CommandExec(java.lang.String)> 能够直接过间接调用到的方法
relation.json 里面是分析得到的所有 <org.joychou.controller.Rce: java.lang.String CommandExec(java.lang.String)> 出发的调用关系。开头项是子项的调用者。
methods.csv 通过处理methods.txt得到，主要为了方便导入neo4j
relation.csv 记录的是relation的csv，为了方便导入neo4j和进行路径搜索的图算法
methods.yaml 可以认为是methods.txt里面的method经过method search筛选过的结果
paths 起点函数和筛选得到的method之间的路径

# --entry-method-yaml-input

name 说明 required default
--entry-method-yaml-input start method file,a yaml true

一个yaml，里面是分析的起点

# --method-feature-yaml-input

name 说明 required default
--method-feature-yaml-input 筛选特定方法,yaml文件 false

call-chain 生成了一个method的集合，放在methods.txt里面，而 --method-feature-yaml-input 如果被设置，则从这些 methods 当中筛选出某些满足特定条件的方法，写入 methods.yml 文件里面。

关于 --method-feature-yaml-input 具体内容的介绍，参考 method搜寻

# --time

name 说明 required default
--time 每一个start method被分析的用时 false 60

# --level

name 说明 required default
--level 调用路径的最大长度 false 7
