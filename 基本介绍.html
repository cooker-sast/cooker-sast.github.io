# 基本介绍

cooker是一个开源的Java程序静态分析工具，专为Java漏洞研究与漏洞挖掘而设计开发。在集成了semgrep之后，可以视为较为完备的sast引擎。

# 特性（Characterization）

  • 启动速度快。将Java程序转储为二进制缓存文件，启动速度快。
  • 分析速度快。支持高并发分析，分析速度快。
  • 分析效果好。基于tai-e IR进行算法设计与编程，经长期漏洞挖掘实践打磨，运行效果又快又准。
  • 覆盖众多文件格式。可以分析多种形式的Java应用，包括但不限于jsp、war、fatjar等。
  • 使用简单。尽量避免使用难懂的参数，分析的结果输出也追求简单易懂，即使没有静态分析背景也可以快速上手。
  • 高级可定制。除了项目自带的规则，使用者还可以根据需求，自定义规则文件，从而实现客制化的分析。

# 基础能力（Basic Features）

能力 是否支持 key 说明
method搜索 ✅ method 支持自定义规则搜索method
class搜索 ✅ class 支持自定义规则搜索class
污点分析 ✅ taint 基于指针分析的污点分析。优化打磨算法细节，适配漏洞挖掘场景

# 高级能力（Advanced Features）

能力 是否支持 key 说明
正向方法调用寻径 ✅ call-chain 支持正向的方法调用寻径。按照程序运行所调用方法的顺序，寻找方法调用路径，使用cha算法
反向方法调用寻径 ✅ invoked-chain 支持反向的方法调用寻径。和正向方法调用寻径的分析思路相反，使用cha算法
全量controller搜集 ✅ controller 搜集Web应用当中的controller，包括但不限于SpringBoot、tomcat、struts2、websocket等
风险controller搜集 ✅ vuln-controller 搜集Web应用当中可能最终会调用危险函数的controller
Web漏洞扫描 ✅ webvuln 一键对一个Web程序进行漏洞扫描
Java反序列化利用链挖掘 ✅ gadget、jnd-gadget 支持对原生反序列化/其他反序列化的链路挖掘

# 设计理念（Design Concept）

  1. 分析原子化，复杂问题分而治之
    • 预设多个原子分析，提升工具的可操作性。可以通过组合多个原子分析解决复杂的问题。
  2. 允许一定程度上的误报来换取覆盖尽可能多的漏洞
    • 程序分析工具无法面面俱到，在漏报和误报的选择上必然要选择误报。如果在使用中发现误报比较严重，可以进行反馈。
  3. 依托于实际的漏洞挖掘与漏洞研究实践
    • cooker的使命是更好地辅助安全研究员挖掘和分析漏洞

# 简易架构（Simple Architecture）

了解cooker的整体架构可以更好地理解cli和配置文件的写法（当然还有可能出现的报错信息），方便大家更好地使用。

# 1. 来源处理（Source Processing）

这一部分是程序分析的入口，cooker会将待分析的Java程序，处理成tai-e IR（中间代码表示），并保存在内存和二进制缓存文件中，用以分析。

目前支持的Java程序的文件格式有以下几种：

应用形式 是否支持 说明
java源文件 ✅ 支持直接分析java源文件，但是一般很少直接用
class文件 ✅ 支持直接分析class文件，但是一般很少直接用
jsp ✅ 支持分析jsp，主要是分析war当中的jsp文件的时候会用到
war ✅ 支持分析一个单独的war文件
exploded war ✅ 支持分析一个exploded war，即war解压之后的文件夹
fatjar ✅ 支持分析SpringBoot fatjar文件
list of jars ✅ 直接分析一个放着众多jar文件的目录（推荐）

# 2. 程序分析（Analysis）

cooker的程序分析引擎是基于IR的，大部分分析的正常运作需要提供规则文件，除了项目自带的规则，使用者还可以根据需求，自定义规则文件，从而实现客制化的分析。

# 3. 结果输出（Output）

cooker通常以文件的形式进行结果的输出，不同分析的具体输出格式参考命令详解。
